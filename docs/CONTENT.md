# Content — sourcing, verification, licensing

**Product:** Amol365 (ইসলামিক আমল)
**Applies to:** everything in `lib/assets/data/`

---

## 1. The rule

> **No Quranic or hadith text ships to users until a qualified human has
> verified it against an authoritative source.**

Every content file carries `_meta.verified`. While that is `false`, the file is
**not release-ready**, no matter how correct it looks.

The reason is specific rather than procedural: this app is used for worship. A
dropped letter changes meaning. A misattributed hadith puts words in the mouth
of the Prophet ﷺ. Neither is a bug that can be patched in the next release —
it will already have been read, memorised, and acted on.

### Why the surah list is short

`surahs.json` deliberately contains only short, universally-memorised passages —
few enough that one person can check every letter in a single sitting.

The long surahs that `FEATURES.md` advertises — **Al-Baqarah (286), Ya-Sin (83),
Ar-Rahman (78), Al-Mulk (30)** — are **intentionally absent**. They must be
ingested from a verified digital edition (see §4), never typed or generated by
hand. Until then the app should not claim to contain them.

### Why there are no hadiths

No verified, permissively-licensed Bangla hadith dataset has been selected yet.
The loader, schema, and daily-rotation logic are complete; only the data is
missing. `hadithListProvider` **drops any entry lacking `source` and
`bookReference`** — unattributed text is a quote, not a hadith, and the app will
not present it as one.

## 3. Verification checklist

Do this before flipping `_meta.verified` to `true`.

**Who:** someone qualified — a hafiz, an imam, or an alim. Not the developer,
and not a proofreader who does not read Arabic.

**Against what:** a printed mushaf, or [tanzil.net](https://tanzil.net) with the
script setting matched to `_meta.script` (currently Imlaei).

- [ ] Every Arabic letter matches, character for character
- [ ] Every diacritic (harakat) matches — including sukun, shadda, madd
- [ ] Verse numbers and counts are correct
- [ ] Verse order is correct
- [ ] Bangla conveys the accepted meaning and introduces no doctrinal claim
- [ ] Transliteration is consistent
- [ ] Surah names, revelation type (Meccan/Medinan), and verse counts are right
- [ ] Ayat al-Kursi is labelled as a passage from Al-Baqarah, not a surah
- [ ] Reviewer's name and date recorded below

**Record it here:**

|---|---|---|---|---|
| `names_of_allah.json` | | | | |
| `surahs.json` | | | | |

## 4. Adding the long surahs

Do **not** hand-type them and do **not** have an AI generate them. Ingest from a
verified edition:

1. Download the Uthmani or Imlaei text from [tanzil.net/download](https://tanzil.net/download/)
2. Download a **permissively-licensed** Bangla translation (see §5)
3. Convert to this app's schema — the shape is in `surahs.json`
4. Verify checksums against the source distribution
5. Spot-check rendering in the app: the Amiri font, RTL layout, and diacritic
   positioning all need eyes on them

Long text also has a practical consequence: Al-Baqarah alone is ~600 KB. Beyond
a few surahs, ship them via the content manifest (M-5 / `docs/SRS-Backend-Auth-Subscription.md`
§9) rather than growing the APK.

## 5. Licensing

**This is a commercial app** (5 BDT/week). Bundling copyrighted text without a
licence is a real legal exposure, not a formality.

| Content | Status |
|---|---|
| Arabic Quran text | Not copyrightable as such. Specific typeset editions may be — use Tanzil's distribution and honour its terms. |
| Bangla Quran translations | **Most are under active copyright.** Popular published translations are *not* free to bundle. |
| Hadith Arabic | Classical, public domain. |
| Bangla hadith translations | Same copyright exposure as above. |
| Current `_meta` content in this repo | Plain-sense renderings written for this app specifically to avoid third-party copyright. Still requires review for accuracy. |

**Before release:** confirm in writing that every bundled translation is either
public domain, permissively licensed, or licensed to you. Record it here:

|---|---|---|---|---|
| | | | | |

## 6. Schema reference

**`names_of_allah.json`**
```json
{ "_meta": { … }, "items": [
  { "number": 1, "arabic": "…", "transliteration": "…", "bangla": "…", "meaning": "…" }
]}
```

**`surahs.json`**
```json
{ "_meta": { … }, "items": [
  { "number": 1, "arabicName": "…", "banglaName": "…", "transliteration": "…",
    "verseCount": 7, "revelationType": "Meccan", "banglaMeaning": "…",
    "isPassage": false, "passageNote": "",
    "ayahs": [ { "number": 1, "arabic": "…", "bangla": "…", "transliteration": "…" } ] }
]}
```

**`hadiths.json`** — `source` and `bookReference` are mandatory; entries without
them are dropped at load.
```json
{ "_meta": { … }, "items": [
  { "id": "bukhari-1", "arabic": "…", "bangla": "…",
    "narrator": "…", "source": "সহিহ বুখারি", "bookReference": "১:১" }
]}
```

Loading is handled by `ContentService` (`lib/app/services/content_service.dart`),
which resolves **downloaded → bundled**, so a future sync (M-5) can update any
of these without an APK release.
